# Yukh Manifesto

## Thesis

A software project is not merely source code and not merely a backlog. It is a changing system of decisions, assumptions, evidence, responsibilities, risks, economic constraints and external commitments.

Yukh exists to make that system explicit, queryable, versioned and projectable.

## Foundational statements

### 1. The project model precedes its tools

GitHub, Jira, Markdown, chat and dashboards expose partial views. None of them is the complete project.

### 2. Memory must be external and governed

Project continuity cannot depend on one person, one chat thread or one agent context window.

### 3. Knowledge is temporal

A decision can be correct at one point and superseded later. Yukh preserves both the current truth and the path that produced it.

### 4. Evidence is distinct from assertion

Designed, implemented, enforced and evidenced are separate states. Projection systems must not collapse them.

### 5. Context is assembled, not copied

An agent or person should receive the smallest coherent context derived from the project model, with provenance and freshness.

### 6. Governance is executable

Rules must become validation, reconciliation, gates and observable drift rather than remain prose alone.

### 7. Economics belongs in the model

Architecture, roadmap and delivery choices produce economic consequences. Yukh must be able to relate decisions and evidence to Economic Units, consumption drivers and Cost-to-Serve models.

### 8. External bodies of knowledge remain external authorities

Yukh connects to UC-BoK, EbD and other sources through traceable adapters; it does not silently copy or redefine their canon.

### 9. Projections are replaceable

GitHub is the first projection, not a permanent architectural dependency of the core model.

### 10. Human responsibility remains visible

Automation may prepare, validate and reconcile. Ownership, approval, risk acceptance and consequential decisions remain attributable to humans.
